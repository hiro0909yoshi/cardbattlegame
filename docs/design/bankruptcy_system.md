# 破産システム

## 概要

EPがマイナスになった際に土地を売却して回復するシステム。

## 発生タイミング

- **ターン終了時**: 通行料支払い後、スキルによる吸魔後
- **ターン開始時**: 敵スペルによる吸魔後（カードドロー後にチェック）

## 処理フロー

### 土地売却で回復可能な場合

1. 「EP不足！土地を売却してください」メッセージ
2. 土地選択UI表示（破産情報パネル + クリーチャー情報）
3. 土地売却 → EP加算
4. EP >= 0 になるまで繰り返し
5. 通常続行（スタートに戻らない）

### 回復不可能な場合（全売却してもマイナス）

1. 全土地を自動売却
2. 「破産した！スタート地点に戻されます」メッセージ
3. スタート地点（タイル0）に移動
4. EP300にリセット
5. プレイヤー刻印クリア（世界刻印はそのまま）
6. 手札は維持

## 土地の売却価格

**売却価格 = 通行料（100%返還）**

※売却のたびに連鎖が変わるため、全土地の通行料表示を再計算

## CPU売却ロジック

**優先順位:**
1. 連鎖がない土地（その属性で1枚だけ）から売却
2. 高額な土地から売却

## 実装ファイル

| ファイル | 役割 |
|---------|------|
| `scripts/game_flow/bankruptcy_handler.gd` | 破産処理のメインロジック |
| `scripts/game_flow_manager.gd` | 破産チェック呼び出し |
| `scripts/player_system.gd` | マイナスEP許容 |
