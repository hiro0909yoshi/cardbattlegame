# スペル視覚演出設計書

**作成日**: 2026-03-07
**ステータス**: 設計中

---

## 概要

スペル・アルカナアーツの使用時に、対象を視覚的に明確にし、効果の適用を演出するシステム。
半透明化（既存）とスペルエフェクト（新規）を組み合わせる。

---

## 現状の仕組み

### 半透明システム（実装済み）

| クラス | 役割 |
|--------|------|
| `TargetMarkerSystem` | `fade_non_target_objects()` / `restore_all_creature_transparency()` 実処理 |
| `TargetSelectionHelper` | マーカー表示/非表示時に上記を呼び出し |
| `creature_card_3d_quad.gd` | `set_transparency(alpha)` カード個別の透明度制御 |

### 現状の問題

- 半透明はターゲット選択中のみ適用、スペル確定後に即復元される
- 全体スペル（`all_creatures` / `all_players` / `world`）では半透明未適用
- ダメージ/回復表示時に半透明なし（どこに効果が出たか視覚的にわかりにくい）

---

## 確定フロー（最終版）

### パターン1: 個別スペル

```
ターゲット選択（対象以外を半透明化 — 既存）
  → ターゲット確定
  → エフェクト再生（確定時、awaitしない）
  → 確定コメント表示（半透明維持）
  → クリック待ち（既存）
  → 効果データ適用
  → 結果表示
  → 半透明復元
```

### パターン2: 全体スペル

```
ターゲット選択（対象外を半透明化 — 追加必要）
  → ターゲット確定
  → 全対象にエフェクト同時再生（確定時、awaitしない）
  → 1体ずつループ:
	→ カメラフォーカス（半透明維持）
	→ コメントのみ（エフェクトなし、半透明維持）
	→ クリック待ち
	→ データ適用 + 結果表示
  → 全体完了後に半透明を完全復元
```

### パターン3: アルカナアーツ

パターン1（個別スペル）と同じフロー。
`SpellMysticArts` 経由で発動するが、半透明+エフェクトの適用方法は共通。

---

## エフェクト実行方針

### 基本原則

- エフェクトは **ターゲット確定時** に再生開始する（確定コメント表示より前）
- エフェクトは **awaitしない**（fire-and-forget、duration指定で自動消滅）
- 確定コメント・クリック待ちなど既存フローは変更しない
- 全体スペルのループ内では **エフェクトなし**（確定時に全対象分を同時再生済み）

### 効果適用中のカメラ

- 効果適用中はカメラ操作を **ロック** する（ドラッグ不可）
- カメラフォーカスはコード側で制御（既存の `focus_camera_on_tile` 等）

### CPUターン

- CPUターンでも確定コメントが表示されるため、エフェクト+半透明は **同様に適用** する
- 将来的に演出速度の調整（`duration_scale`）を検討

## スペルエフェクト（新規 — 後続実装）

### エフェクトの種類（案）

| エフェクト | 用途 | 演出案 |
|-----------|------|--------|
| ダメージ | 攻撃スペル | 赤い衝撃波 / 属性別パーティクル |
| 回復 | 回復スペル | 緑の光 / 上昇パーティクル |
| バフ | ステータス上昇 | 青い光の柱 / 上昇矢印 |
| デバフ | ステータス下降 / 刻印 | 紫の霧 / 下降矢印 |
| 移動 | プレイヤー/クリーチャー移動 | 移動軌跡エフェクト |
| 属性変化 | 土地属性変更 | 属性色への変色エフェクト |
| 汎用 | その他 | 白い光のフラッシュ |

### エフェクトクラス設計（案）

```
SpellVisualEffect（新規）
  - play_effect(tile_index, effect_type, element) → エフェクト再生
  - stop_effect() → エフェクト停止
  - is_playing() → 再生中判定

SpellVisualManager（新規）
  - 半透明の開始/復元タイミング管理
  - エフェクト再生の統合制御
  - SpellEffectExecutor / SpellDamage から呼び出される
```

### 注意点

- エフェクトのライフサイクル管理が必要（前のエフェクトが残っている場合の処理）
- GPU負荷を考慮し、同時表示エフェクト数に上限を設ける（全体スペルで大量発生防止）
- エフェクトは自動消滅（duration指定）とし、明示的なstopは不要にする

## 半透明タイミングの変更点

### 現状

```
[ターゲット選択開始] → fade開始
[マーカー非表示]     → restore（ここで復元されてしまう）
[効果適用]           → 半透明なし
```

### 変更後

```
[ターゲット選択開始] → fade開始
[ターゲット確定]     → エフェクト再生（半透明維持）
[マーカー非表示]     → マーカーのみ非表示（半透明は維持）
[確定コメント表示]   → 半透明維持
[効果適用]           → 半透明維持
[結果表示]           → 半透明維持
[全処理完了]         → restore（ここで初めて復元）
```

### 安全復元（フェイルセーフ）

スペルがキャンセル・失敗した場合に半透明が残らないよう、
`SpellEffectExecutor` の全終了パス（成功・失敗・キャンセル）で必ず `restore` を呼ぶ。

## 実装順序

### Phase 1: 半透明タイミング修正（優先）
1. マーカー非表示と半透明復元を分離（`_hide_instance_marker` / `hide_selection_marker`）
2. 個別スペル: 確定→効果適用完了まで半透明維持
3. `SpellEffectExecutor` の全終了パスで `restore` を呼ぶ（フェイルセーフ含む）
4. 効果適用中のカメラロック

### Phase 2: 全体スペル対応
1. `show_confirmation_highlights()` に半透明追加（`all_creatures` / `all_players` / `world`）
2. `SpellDamage` の全体処理ループで半透明維持
3. 複数対象用 `fade_non_target_tiles()` 新設

### Phase 3: スペルエフェクト（後続）
1. `SpellVisualEffect` クラス新設（基本エフェクト）
2. ターゲット確定時にエフェクト再生を呼び出す仕組み
3. ダメージ/回復の汎用エフェクト実装
4. 属性別エフェクトの追加

### Phase 4: 統合・調整（後続）
1. `SpellVisualManager` で半透明+エフェクトの統合管理
2. アルカナアーツへの適用
3. CPUターン時の演出速度調整（`duration_scale`）

## 議論メモ（2026-03-07）

- エフェクトのタイミング: 確定コメント時ではなく **ターゲット確定時** に再生
- 全体スペルのループ内: エフェクトなし、コメント+半透明のみ
- エフェクトはawaitしない（fire-and-forget）
- 効果適用中はカメラ操作ロック
- CPUターンでも確定コメントがあるためエフェクト+半透明は同様に適用
- 実装優先順: 半透明タイミング修正（Phase 1-2）→ エフェクト（Phase 3-4）
- fire-and-forgetエフェクトで「固まり」は解消されない（GPU負荷が増えるだけ）
- 固まり対策は別途: フレーム分散 / 一括適用+まとめ表示 / CPU自動送り
