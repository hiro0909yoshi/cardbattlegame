# メイン画面設計書

## 概要

ゲーム起動後のホーム画面。全機能への導線を提供する。

---

## 背景システム

### ノード構成

```
Background（最背面、全画面）
├── BackgroundImage (TextureRect)     -- メイン背景画像
│   stretch_mode = KEEP_ASPECT_COVERED
│   画像差し替えだけで雰囲気を変更可能
│
├── OverlayGradient (ColorRect)       -- 半透明オーバーレイ
│   Color: rgba(0, 0, 0, 0.3~0.5)
│   UIテキストの可読性を確保
│   左パネル部分はやや濃くする等の調整可能
│
└── ParticleEffect (GPUParticles2D)   -- パーティクル演出（任意）
    光の粒、魔法陣の回転等
    オプション（なくても成立する）
```

### 背景画像の仕様

| 項目 | 値 |
|------|-----|
| 推奨サイズ | 1920×1080px 以上 |
| アスペクト比 | 16:9（KEEP_ASPECT_COVERED で自動調整） |
| 形式 | PNG or WebP |
| 配置場所 | `assets/images/ui/main_menu_bg.png` |

### 背景のバリエーション（将来）

イベントや季節に応じて背景を差し替え可能な構造にしておく。

| パターン | 用途 |
|---------|------|
| 通常背景 | デフォルト |
| イベント背景 | 大会開催中、季節イベント等 |
| 特別背景 | 周年記念等 |

背景画像パスを設定や条件で切り替えるだけで対応可能。

```gdscript
# 例: イベント背景の切り替え
var bg_path = "res://assets/images/ui/main_menu_bg_event.png"
background_image.texture = load(bg_path)
```

## エリア構成

### 1. 上部バー（リソース・通知）

背景色 `Color(0.15, 0.15, 0.15, 0.9)`（黒とグレーの中間、少し透過）。
左側にリソース＋追加ボタン、右側に通知・設定アイコンを配置。フォントサイズ50。

#### 左側（リソース）

| アイコン | ラベル | ＋ボタン | 説明 |
|---------|--------|---------|------|
| ⚡ | StaminaLabel | StaminaPlusButton | スタミナ `現在値/最大値` |
| 💰 | GoldLabel | GoldPlusButton | 所持ゴールド |
| 💎 | StoneLabel | StonePlusButton | 所持課金石 |

各リソース間にスペーサー（40px）を配置。

#### 右側（通知・設定）

| アイコン | ボタン名 | タップ時 |
|---------|---------|---------|
| 📋 | DailyQuestButton | デイリークエスト画面へ |
| 📢 | AnnouncementButton | お知らせ一覧へ |
| 📧 | MailButton | メール受信箱へ |
| ⚙️ | SettingsButton | 設定画面へ (`Settings.tscn`) |

アイコン間隔: 40px。

画面左側（幅比率 4:6 で左が4）。パネル全体が `gui_input` でタップ検出可能で、ステータス画面へ遷移する。

| 項目 | ノード | 説明 |

| プレイヤー名 | PlayerNameLabel (font48) | `GameData.player_data.profile.name` |

| キャラクター | CharacterRect (300×400) | アバター画像 |
| ゴールドリセット | ResetGoldButton | デバッグ用（後で削除） |

**タップ時**: プレイヤーステータス画面へ遷移（未実装）

### 3. BattleContainer（バトル系ボタン・中央）

ゲームのメインコンテンツへの導線。4つの大型ボタン（各700×300, font72）を2×2グリッド状に配置。
行間120px、列間160px。コンテナは `size_flags_vertical = expand` で中央に配置。

#### 上段

| ボタン | 説明 | 遷移先 |
|--------|------|--------|
| クエスト | ストーリー進行型PvE。スタミナ消費 | `WorldStageSelect.tscn` |
| 大 会 | 期間限定トーナメント/リーグ戦 | 未実装 |

#### 下段

| ボタン | 説明 | 遷移先 |
|--------|------|--------|
| ネット対戦 | オンライン1vs1。レーティング制 | 未実装 |
| ソロバトル | CPU対戦（練習用）。報酬なし | `Album.tscn`（バトルモード） |

※クエスト・ネット対戦・大会・ソロバトルはデッキ検証を行い、所持していないカードがデッキにある場合は警告ダイアログを表示。

サブ機能への導線。右寄せ（alignment=2）で配置。下マージン40px、右マージン40px。

| ボタン | サイズ | フォント | 説明 | 遷移先 |
|--------|--------|---------|------|--------|
| アルバム | 140×140 | 22 | カード一覧・デッキ編集 | `Album.tscn` |
| 倉庫 | 140×140 | 22 | 所持アイテム管理・使用 | 未実装 |
| 施設 | 140×140 | 22 | 施設関連機能 | 未実装 |

アルバム・倉庫・施設は `size_flags_vertical = 8`（Shrink End）で、ショップの下端に揃えて配置。

## プレイヤーレベルシステム

### 概要
経験値を貯めてレベルアップ。汎用的な仕組みとし、経験値源は後から追加可能にする。

### 経験値の獲得源

| 行動 | 経験値 | 備考 |
|------|--------|------|
| クエストクリア | 難易度による | メイン経験値源 |
| デイリークエスト達成 | 各ミッションごと | 毎日コツコツ |
| 大会参加/勝利 | 要調整 | 勝利ボーナスあり |
| （将来追加枠） | - | 汎用設計で拡張可能 |

※ネット対戦、ソロバトルでの経験値は今後検討

### レベルアップ報酬

| 報酬 | 説明 |
|------|------|
| スタミナ全回復 | レベルアップ時に即回復 |
| スタミナ上限UP | 一定レベルごとに最大値増加 |
| （将来追加枠） | コンテンツ解放等 |

### 実装方針
- 経験値追加は `add_experience(amount, source)` のような汎用メソッド
- `source` パラメータでどこから獲得したか記録（分析用）
- レベルテーブルはJSON等のデータ駆動で管理

### デフォルトマップ

### データ管理
- ユーザーごとに獲得済みマップIDリストを保存

## 課金石システム

### 概要
リアルマネーで購入するプレミアム通貨。詳細な用途は今後拡充。

### 初期の用途

| 用途 | 説明 |
|------|------|
| プレミアムガチャ | 通常ガチャより排出確率が良い |
| スタミナ全回復 | 課金石消費で即回復 |

### ゴールドとの使い分け
- **ゴールド**: ゲームプレイで獲得。通常ガチャ、通常ショップ購入
- **課金石**: 主にリアルマネー購入。高確率ガチャ、スタミナ回復、将来的に限定コンテンツ

## 大会システム

### 概要
期間限定の競技イベント。エントリー費用なし。

### 形式（暫定）
リーグ戦 → 上位リーグ同士で対戦 → 最終的にトーナメントで頂点決定

```
[リーグ予選]
  グループA: 8人総当たり → 上位2名通過
  グループB: 8人総当たり → 上位2名通過
  ...
     ↓
[決勝トーナメント]
  リーグ通過者によるトーナメント戦
     ↓
  優勝決定
```

### レーティング
- リーグ内の順位はレーティングで決定
- 勝利でレーティング上昇、敗北で下降

### 報酬
- 上位入賞: 限定称号、限定カード、課金石等
- 参加賞: ゴールド、経験値

### 詳細設計
- 大会の具体的なマッチング人数、期間、開催頻度等は今後決定
- ユーザー規模に応じて調整

## 称号システム

### 概要
実績達成やイベント報酬で称号を獲得。1つを「装備」してプレイヤー名の下に表示。

### 称号カテゴリ

| カテゴリ | 例 | 解放条件例 |
|---------|-----|-----------|
| 対戦系 | 「百戦錬磨」「不敗の王」 | 通算100勝、10連勝 |
| コレクション系 | 「カードコレクター」「完全図鑑」 | 50種収集、全カード収集 |
| クエスト系 | 「冒険者」「英雄」 | ワールド1クリア、全クエストクリア |
| 属性系 | 「炎の支配者」「大地の守護者」 | 火属性カード全収集等 |
| イベント系 | 「初代チャンピオン」 | 大会優勝（期間限定） |
| 特殊 | 「はじまりの一歩」 | チュートリアル完了 |

## SNS連携

### 概要
ゲーム内の成果をSNSに共有し、プレイヤーの発信を促進する。
URLスキーム方式で実装し、将来的にネイティブSDK連携に移行可能な構造とする。

### 連携先

| SNS | 方式 | 備考 |
|-----|------|------|
| X (Twitter) | Web Intent URL (`https://twitter.com/intent/tweet?text=...`) | テキスト+画像。拡散力が高い |
| Discord | Webhook投稿 + クリップボードコピー | 自前サーバーへの投稿。コミュニティ連携 |
| LINE | URLスキーム (`line://msg/text/...`) | 日本ユーザー向け。友達への共有・招待 |
| Instagram | 画像保存 → アプリ起動 | カード画像向き。将来ネイティブSDK対応 |

### 共有タイミングと共有内容

#### ステータス画面内

| タブ | 共有内容 | 画像 |
|------|---------|------|
| 戦績 | 戦績サマリー（ランク、勝率、連勝記録） | 戦績カード画像 |
| 実績 | 実績達成報告（「称号『不敗の王』を獲得！」） | 実績カード画像 |
| マップ | マップクリア報告 | マップサムネイル |
| キャラ | プロフィールカード（アバター+称号+お気に入りカード） | プロフィールカード画像 |

#### ゲームプレイ中

| タイミング | 共有内容 | 画像 |
|-----------|---------|------|
| 対戦勝利 | 勝利報告（対戦相手、スコア） | 勝利リザルト画面 |
| レアカード入手 | カード入手報告 | カード画像 |
| 大会入賞 | 大会結果（順位、大会名） | 入賞カード画像 |

### 共有画像生成

Godotの`SubViewport`で共有用画像を動的生成する。

- **サイズ**: 1200×630px（OGP推奨サイズ、各SNSで見切れにくい）
- **内容**: ゲームロゴ + 共有内容 + プレイヤー情報
- **形式**: PNG保存 → 各SNSに添付
- **ブランディング**: ゲームタイトル「Arcana Conquest」のロゴとURLを常に含める

### 実装方式

```
[共有ボタンタップ]
  ↓
[共有画像を SubViewport で生成 → PNG保存]
  ↓
[SNS選択ダイアログ表示]
  ├─ X → OS.shell_open("https://twitter.com/intent/tweet?text=...")
  ├─ Discord → HTTPRequest でWebhook送信 or クリップボードコピー
  ├─ LINE → OS.shell_open("line://msg/text/...")
  └─ Instagram → 画像をギャラリー保存 → アプリ起動
```

### 共有テキストテンプレート例

```
Arcana Conquest
ランクマッチ100勝達成！
称号「百戦錬磨」を獲得しました！

#ArcanaConquest #アルカナコンクエスト
```

### 外部設定（手動対応が必要）

| 作業 | 内容 |
|------|------|
| X開発者アカウント | Web Intent方式なら不要。API連携する場合は要登録 |
| Discord Bot/Webhook | サーバーの Webhook URL を設定画面で管理 |
| LINE URLスキーム | 追加設定不要（標準スキーム） |
| Instagram | 画像保存のみなら追加設定不要 |
