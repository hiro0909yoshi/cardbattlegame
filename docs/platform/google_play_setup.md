# Google Play Developer アカウント・リリース手順

## デベロッパーアカウント

| 項目 | 内容 |
|------|------|
| アカウント種別 | 個人 |
| デベロッパー名 | Katsura Studio |
| 登録費用 | $25（一回のみ） |
| 本人確認 | 免許証写真の提出が必要 |

> **注意**: 本人確認が承認されるまでアプリのアップロードができない場合がある

## クローズドテストに必要なもの

### 必須

- [ ] AABファイル（ビルド済み）
- [ ] プライバシーポリシーのURL
- [ ] ストア掲載情報
  - アプリ名
  - 短い説明（80文字以内）
  - 詳しい説明
  - アプリアイコン（512x512 PNG）
  - フィーチャーグラフィック（1024x500 PNG）
  - スクリーンショット 2枚以上
- [ ] コンテンツレーティング質問票（Play Console内で回答）
- [ ] ターゲットユーザー層の設定（Play Console内で回答）

### テスター管理

- クローズドテストではメールアドレスでテスターを招待
- テスターはGoogle Playからアプリをインストール
- 最大テスター数: 制限なし（クローズドテスト）

## リリースフロー

```
クローズドテスト → オープンテスト（任意） → 製品版リリース
```

### クローズドテスト手順

1. Google Play Console にログイン
2. アプリを作成（アプリ名、デフォルト言語、アプリ/ゲーム、無料/有料）
3. ストア掲載情報を入力
4. コンテンツレーティング質問票に回答
5. プライバシーポリシーURLを設定
6. テスト → クローズドテスト → トラックを作成
7. AABファイルをアップロード
8. テスターのメールリストを追加
9. 審査に提出

## プライバシーポリシー

- GitHub Pages でホスティング（`docs/privacy_policy.html`）
- URLパターン: `https://<username>.github.io/<repo>/privacy_policy.html`
- 正式リリース時に内容を充実させる

## 注意事項

- **キーストア管理**: 紛失するとアプリ更新不可。バックアップ必須
- **Play App Signing**: Google Play側でも署名管理される（推奨設定）
- **バージョンコード**: アップデート時は必ずインクリメントする
- **ターゲットSDK**: Google Playの要件に合わせて更新が必要
